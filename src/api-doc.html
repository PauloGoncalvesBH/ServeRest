ServeRest

Servidor exemplo de um e-commerce com intuito de servir de material de estudos de testes de API.

Ajuda ¶

Precisa de ajuda? Abra uma issue em https://github.com/PauloGoncalvesBH/serverest

Comandos do terminal ¶

Comando Descrição Default Exemplo
-p ou --porta Porta que será utilizada 3000 npx serverest -p 3500
-t ou --timeout Timeout do token de autenticação em milissegundos 1000 npx serverest -t 100000
-n ou --nodoc Desabilitar o início automático da documentação npx serverest --nodoc
-h ou --help Exibir ajuda no terminal npx serverest --help

Autenticação ¶

O token de autenticação é adquirido através de POST na rota /login e deve ser incluído no Header das rotas que requerem autenticação.

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJlbWFpbCI6ImZ1bGFub0BxYS5jb20iLCJwYXNzd29yZCI6InRlc3RlIi

Rotas que necessitam de autenticação ¶

Note que determinadas rotas necessitam que o usuário seja administrador

"administrador": "true"

Verbo Rota Usuário administrador
POST /produtos Sim
PUT /produtos Sim
DELETE /produtos Sim
POST /carrinhos Não
DELETE /carrinhos/concluir-compra Não
DELETE /carrinhos/cancelar-compra Não

Filtrar retorno de GET ¶

As requisições GET retornam todos os dados por padrão. Para filtrar o resultado utilize query string.
